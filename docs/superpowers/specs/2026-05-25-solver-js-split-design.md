# `solver.js` per-puzzle split — design

Date: 2026-05-25
Status: approved

Splits the ~11k-line `solver.js` monolith into one file per puzzle
solver. Cuts per-edit context window for any single solver from
"whole file" to "~600-1000 lines". Preserves the existing runtime
shape: a single concatenated `dist/solver.js` is still what the
extension loads, fetched as text and stitched into the Blob worker by
`content.js`.

## 1. Goal

- Move each `*Solver` class into its own source file so editing one
  puzzle no longer requires scrolling through 11k lines.
- Keep Node tests, the extension content script, and the inlined Blob
  worker working unchanged.
- No bundler dependency — preserve the project's "no build tooling"
  philosophy. A ~30-line Node script handles the concatenation.

## 2. Layout

```
src/solvers/
  nonogram.js             NonogramSolver
  aquarium.js             AquariumSolver
  galaxies.js             GalaxiesSolver
  binairo.js              BinairoSolver
  shikaku.js              ShikakuSolver
  yinyang.js              YinYangSolver
  slitherlink.js          SlitherlinkSolver
  hashi.js                HashiSolver
  heyawake.js             HeyawakeSolver
  hitori.js               HitoriSolver
  kakurasu.js             KakurasuSolver
  kurodoko.js             KurodokoSolver
  mosaic.js               MosaicSolver
  norinori.js             NorinoriSolver
  nurikabe.js             NurikabeSolver
  diff.js                 computePuzzleDiff
  index.js                Node-side re-exports
scripts/
  build-solver-bundle.js  concatenation tool
solver.js                 1-line shim → src/solvers/index.js
dist/solver.js            generated by build-solver-bundle.js
```

The repo-root `solver.js` becomes a 1-line CommonJS shim so the
~30 test/bench files that `require('../solver.js')` keep working
unchanged. The shipped bundle (`dist/solver.js`) is the
single-concatenated artifact the extension loads.

## 3. Per-puzzle file format

Each file is a standalone CommonJS module:

```js
'use strict';

class NonogramSolver {
  // ... existing class body, verbatim ...
}

if (typeof module !== 'undefined' && module.exports) {
  module.exports = { NonogramSolver };
}
```

Three properties matter:

- **'use strict'** at the top. Each file is independently loadable.
- **Class declaration** at top level — no IIFE wrapper. The class
  name becomes a global identifier inside the concatenated bundle.
- **Conditional CJS export** at the bottom. The export block is what
  the build script strips; the class declaration above is what it
  keeps and concatenates.

`diff.js` follows the same pattern, exporting `{ computePuzzleDiff }`.

## 5. Build script — `scripts/build-solver-bundle.js`

```js
'use strict';
const fs = require('fs');
const path = require('path');

// Order matters only insofar as readability — classes don't reference
// each other at file scope, so alphabetical is fine. `diff.js` last
// because `computePuzzleDiff` is the cross-puzzle helper.
const FILES = [
  'aquarium.js', 'binairo.js', 'galaxies.js', 'hashi.js',
  'heyawake.js', 'hitori.js', 'kakurasu.js', 'kurodoko.js',
  'mosaic.js', 'nonogram.js', 'norinori.js', 'nurikabe.js',
  'shikaku.js', 'slitherlink.js', 'yinyang.js',
  'diff.js',
];
const EXPORTS = [
  'NonogramSolver', 'AquariumSolver', 'GalaxiesSolver', 'BinairoSolver',
  'ShikakuSolver', 'YinYangSolver', 'SlitherlinkSolver', 'HashiSolver',
  'HeyawakeSolver', 'HitoriSolver', 'KakurasuSolver', 'KurodokoSolver',
  'MosaicSolver', 'NorinoriSolver', 'NurikabeSolver', 'computePuzzleDiff',
];

// Strip the trailing CommonJS export block:
//   if (typeof module !== 'undefined' && module.exports) { ... }
// Matches the LAST occurrence so a class body containing the literal
// string can't fool us.
const EXPORT_RE =
  /\n\s*if\s*\(\s*typeof\s+module[\s\S]*?module\.exports\s*=\s*\{[\s\S]*?\}\s*;?\s*\}\s*$/;

const srcDir = path.join(__dirname, '..', 'src', 'solvers');
const parts = [
  "'use strict';",
  "// Generated by scripts/build-solver-bundle.js — do not edit by hand.",
  "",
];
for (const file of FILES) {
  const fullPath = path.join(srcDir, file);
  let body = fs.readFileSync(fullPath, 'utf8');
  // Drop the per-file 'use strict' to avoid duplication.
  body = body.replace(/^\s*'use strict';\s*\n/, '');
  // Drop the per-file CommonJS export block.
  body = body.replace(EXPORT_RE, '\n');
  parts.push(`// ── ${file} ──`);
  parts.push(body.trim());
  parts.push("");
}
parts.push("if (typeof module !== 'undefined' && module.exports) {");
parts.push(`  module.exports = { ${EXPORTS.join(', ')} };`);
parts.push("}");

const outDir = path.join(__dirname, '..', 'dist');
fs.mkdirSync(outDir, { recursive: true });
fs.writeFileSync(path.join(outDir, 'solver.js'), parts.join('\n'));
```

The script aborts (throws) on missing files. It assumes each input
ends with the standard CommonJS export block; if the regex doesn't
match a file, the script throws an error rather than silently
producing a broken bundle.

## 6. `package.json` build script change

Current:

```
"build": "rm -rf dist && mkdir -p dist/icons && cp manifest.json
  background.js main-world.js content.js handler.js solver.js
  solver.worker.js dist/ && cp icons/icon-16.png icons/icon-32.png
  icons/icon-48.png icons/icon-128.png dist/icons/"
```

New:

```
"build": "rm -rf dist && mkdir -p dist/icons && node
  scripts/build-solver-bundle.js && cp manifest.json background.js
  main-world.js content.js handler.js solver.worker.js dist/ &&
  cp icons/icon-16.png icons/icon-32.png icons/icon-48.png
  icons/icon-128.png dist/icons/"
```

The `cp solver.js dist/` is replaced by the bundle script (which
writes directly to `dist/solver.js`). `solver.js` at root is the shim
and never needs to be copied.

## 7. Root `solver.js` shim

```js
'use strict';
module.exports = require('./src/solvers/index.js');
```

Keeps `tests/**/*.test.js`, `tests/bench-*.js`, and any other
Node-side caller working with their existing `require('../solver.js')`
imports.

## 8. Tests

No new tests. Existing suite must pass before and after. To catch
bundle-vs-source drift, add one integration check inside the build
script's CI invocation:

```
node -e "require('./dist/solver.js'); console.log('bundle loads');"
```

If the bundle is malformed (e.g. an export was missed by the regex),
this `require` throws and the build fails loudly.

## 9. Migration steps

1. Create `src/solvers/` and `scripts/` directories.
2. For each existing solver class, extract it into its own file —
   copy the class declaration verbatim, add `'use strict';` + CJS
   export footer. Confirm via `node -e "require('./src/solvers/foo.js')"`.
3. Extract `computePuzzleDiff` into `src/solvers/diff.js`.
4. Write `src/solvers/index.js`.
5. Replace the contents of root `solver.js` with the 1-line shim.
6. Add `scripts/build-solver-bundle.js`.
7. Update `package.json` build script.
8. Run `npm test` — must show identical pass count to pre-refactor
   (currently 448).
9. Run `npm run build` — `dist/solver.js` produced; verify it loads.
10. Run `npm run lint` and `npm run typecheck` — clean.

## 10. Risks

- **Hidden cross-class references**: confirmed none exist at file
  scope today. If any are introduced later, the bundler order in
  `FILES[]` may need adjustment, or the offending reference moved into
  a shared helper.
- **Static initialization order across files**: each solver's static
  fields (`_solutionCache`, `_maxSolutionCache`, etc.) are
  self-contained. Order between solvers does not matter.
- **Regex over-matches** the CommonJS export block: covered by the
  build-time `require('./dist/solver.js')` check; a malformed bundle
  fails the build.
- **Two files named `solver.js`** (root shim + `dist/`): noted, lived
  with. The shim is one line so accidental confusion is bounded.
  Rename is a future option, not part of this refactor.

## 11. Out of scope

- Extracting shared utilities (FNV-1a, trail-based undo helpers) into
  a common module — YAGNI. Each solver keeps its current inlined
  copy.
- Switching to ES modules or adding esbuild — project keeps the
  bundler-free philosophy.
- Splitting `content.js` (~4.5k LOC) — a separate effort.
- Tooling around per-solver test selection (`npm test --grep`) — the
  existing `npm test` already runs the whole node:test suite quickly.

End of design.
